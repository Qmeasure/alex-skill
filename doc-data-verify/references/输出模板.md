# 输出模板

`doc-data-verify` 的两份交付物:核查表 + 总结,和直接改干净的修正稿。这份定字段、定格式、定改稿规则。

## 一、核查表

6 个字段,一项数据一行。

| 字段 | 填什么 |
|---|---|
| 原文数据 | 文稿里的原始表述,照抄(含数值、单位、口径词),并带文中位置(第几段 / 哪句) |
| 核查结果 | 五态之一:一致 / 不一致 / 已过期 / 缺失 / 冲突 |
| 最新数据 | 核实到的正确值(含单位口径)。状态=一致时填"同原文";缺失/冲突按下方规则填 |
| 来源URL | 可点开的一手源链接;多源用换行或顿号列出。必须是真实 URL,不许写"公司公告""网络资料"这种笼统词 |
| 数据截止时间 | 该数据的发布 / 统计截止时间(如 2025Q3、2025-12-31、2024 年报) |
| 备注说明 | 信源等级、口径差异、单源置信度、为何过期/冲突、检索路径等 |

### markdown 模板

```markdown
| 原文数据 | 核查结果 | 最新数据 | 来源URL | 数据截止时间 | 备注说明 |
|---|---|---|---|---|---|
| 营收 11.74 亿元(第2段) | 一致 | 同原文 | https://… 公司2024年报 | 2024 年报 | 一手源,口径对 |
| 市占率 23%(第3段) | 已过期 | 18.5% | https://… 行业协会2025 | 2025H1 | 原文用的是 2023 年数 |
| 全球排名第 4(第1段) | 不一致 | 第 6 | https://… IDC 2025Q2 | 2025Q2 | 口径:按出货量;原文未注口径 |
| 对标公司 B 的营收 | 缺失 | 8.2 亿元 | https://… B公司2024年报 | 2024 年报 | 原文只给了 A,B 未提,补齐对比组 |
| 用户数 5000 万(第4段) | 冲突 | 4800 万 / 5200 万 | https://…官方 ; https://…财报 | 2024 年报 | 两个一手源不一致,待用户定 |
```

### 五态判定标准

- **一致** —— 原文与权威源吻合:数值、单位、口径、时间都对。
- **不一致** —— 原文与核实值不符。写错数、用错单位、口径错(如把 GMV 当营收)。
- **已过期** —— 原文曾经对,但已有更新数据。最新数据列填新值,备注写原文用的是哪期。
- **缺失** —— 文稿该有却没有的同类数据。最重要的场景是对比/系列里缺的那一项——补搜到,填进最新数据列,备注标"补齐对比组"。
- **冲突** —— 多个权威源给出不同值,无法唯一确认。最新数据列并列各值,来源列并列各源,交用户定。

## 二、总结段

核查表后跟一段总结,三层:

1. **整体准确性** —— 共核查 N 项,一致 X 项 / 需更新 Y 项(不一致+已过期)/ 缺失 Z 项 / 冲突 W 项。一句话给文稿数据的整体可信度。
2. **主要问题** —— 挑出最影响结论的几处错/过期/缺口,讲清错在哪、影响什么判断。
3. **建议修改方向** —— 哪些已在修正稿改好;哪些(冲突/缺失)需用户决策才能定;时效上哪些数据建议等新一期披露再更新。

## 三、修正稿(直接改干净)

把能**唯一确认**的错值、过期值,在文稿里就地替换成正确值。

规则:

- **干净替换。** 正文只留正确值。不加 `[已更正]`、不加脚注、不写"原为 X"、不留来源标注。读者看到的是一份正常文稿,不是带批注的校对稿。
- **来源只在表里。** 所有 URL、截止时间、信源等级都在核查表,正文正文不掺。
- **冲突与缺失不动。** 状态=冲突 或 文稿原值对应「未找到公开数据」的,**保留原文不改**,只在核查表标出。不猜值、不替用户选。
- **缺失项的补齐**进核查表即可,是否要把补到的对比数据写进正文,交用户定(除非用户明确要求补写)。

### 落地方式

- **磁盘文件**(md/txt/docx):先备份(`原名.bak` 或带时间戳副本),再用 Edit 就地覆盖原文件,改完报告备份位置和改了哪几处(改动清单指向核查表,不在正文留痕)。
- **粘贴文本**:直接输出改好的干净全文,不碰磁盘。

docx 的读写按需用 `docx` skill 处理,纯文本/markdown 直接 Edit。
