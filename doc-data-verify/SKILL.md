# 文稿数据核实与更新(doc-data-verify)

给定一份文稿,把里面每一个数字、日期、比率、排名都拎出来联网核实,标清楚对不对、过没过期,产出一张核查表,再把能确认的错值就地改干净。

核验纪律(零记忆、多源交叉、信源分级、数量级合理性、停止信号)与 `verify-before-answer` 同源,细则见那份和 `references/信源与核验.md`,这里不重抄。

## 核心原则

三条底线,贯穿全流程,违反任一条都要重做。

**零记忆依赖。** 不许拿训练记忆当事实依据。文稿里每一个数据点,都要实时联网搜出来,并附可点开的来源 URL。心里"记得是多少"不算核实——查到、读到、对上来源,才算。

**完整性强制。** 不许输出空白、占位符、模糊表述。某项数据多轮搜索仍拿不到,就明确写「未找到公开数据」,并交代检索路径(搜了什么关键词、看了哪些源),而不是留白或用"大概""估计"糊过去。核查表里没有空单元格。

**同类对齐。** 文稿涉及类比、对比、系列数据时(比如"A 公司 vs B 公司""近五年趋势"),所有对比项都要取到,且统计口径和时间范围一致——同是年度数、同币种、同截止年。缺一项对比项,或口径不齐,整组结论不成立,补齐再下判断。

## 执行流程

五步,按序走完。

**一、数据提取。** 通读文稿,把所有事实性信息识别出来:数值、金额、百分比、日期、排名、份额、增速、人名职位、机构数据。建一张待核查清单,每项记下原文表述和它在文中的位置。

**二、联网验证。** 对清单里每一项执行实时搜索。优先采信权威一手源:政府公报、上市公司财报、官方统计机构、监管文件、学术论文原文。一手源拿不到再退而求其次,并在备注里标信源等级。关键数据(影响主结论的)要 ≥2 个独立来源互证。

**三、时效校验。** 确认每个数据的发布/统计时间。文稿要的是"最新"但只搜到历史数据时,不许拿旧数充新——标清楚数据截止时间,并提示用户"现有公开数据更新到 X,更新的暂未发布"。

**四、交叉比对。** 把搜索结果与文稿原文逐项对照,每项标一个状态:

- **一致** —— 原文数据与权威源吻合(口径、时间、量级都对)。
- **不一致** —— 原文与核实值不符(写错、口径错、单位错)。
- **已过期** —— 原文曾经对,但已有更新的数据。
- **缺失** —— 文稿该有却没有的同类数据(尤其对比组里缺的那项)。
- **冲突** —— 多个权威源给出不同值,无法唯一确认。

**五、补全与修正。** 不一致/已过期的,给出更新值 + 来源;缺失的同类数据,补搜到整组齐全;冲突的,列出各版本和信源。全部落进核查表。

## 输出

两份交付物,格式细则见 `references/输出模板.md`。

**1. 核查表 + 总结。** 6 字段表格:原文数据 | 核查结果(五态之一)| 最新数据 | 来源URL | 数据截止时间 | 备注说明。表后跟总结段:整体准确性、主要问题、建议修改方向。

**2. 修正稿(直接改干净)。** 把能唯一确认的错值/过期值在文稿里**就地替换**成正确值。正文保持干净——不加改动标注、不留旧值、不写"这里原来错在哪"。来源痕迹只在核查表里,文稿正文不掺。

落地方式两种,自动判别:

- 文稿是磁盘文件(md/txt/docx)→ 用 Edit 就地覆盖原文件。**动手前先备份一份**(如 `原名.bak`),改完告诉用户备份在哪。
- 用户粘贴的文本 → 直接输出改好的干净版,不碰磁盘。

**冲突与缺失项不自动改。** 冲突值(多源不一致)和「未找到公开数据」的项,**保留文稿原值不动**,只在核查表标出候选与信源,等用户拍板。不猜、不编、不替用户做决定。

## 异常处理

**多个冲突值。** 搜出来几个权威源各执一词,就把各版本和对应信源都列在核查表(状态=冲突),由用户判断采纳哪个。不自作主张选一个填进正文。

**非公开 / 付费数据。** 某项数据属于非公开或付费墙后内容(如未披露的私有公司营收、收费数据库数字),明确告知"该数据非公开",并给一个可替代的公开指标建议(如用行业协会口径、可比上市公司、官方抽样替代)。

**输出前自检(强制门)。** 每次交付前过两道,任一不过就回去重查,不交付:

1. 跑 `python scripts/verify_lint.py <核查表文件>` —— 机械检查空白、占位符、非法状态、缺更新值/来源/截止时间、记忆型对冲词。命中即非零退出,逐条清掉。
2. prose 自检清单(下方),对照"零记忆依赖"和"完整性强制"逐条过。

## 自检清单(prose)

交付前对照,有一条不满足就回去补:

- 每个进核查表的数据,是不是都来自这次实时搜索、带了可点开的 URL?有没有哪个是"凭印象"填的?
- 核查表有没有空单元格、占位符、"待补/暂无/大概"?完整性强制不许留白——查不到要写「未找到公开数据」+ 检索路径。
- 每项的核查结果,是不是 {一致/不一致/已过期/缺失/冲突} 之一?
- 状态是不一致或已过期的,有没有给出最新数据 + 来源 URL + 数据截止时间?
- 对比/系列数据,所有对比项是不是都取到了、口径和时间范围对齐?缺项标了「缺失」没有?
- 时效敏感的数据(价格、市值、最近财报、排名),是不是当次搜的、标了截止时间?
- 冲突值和未找到的项,有没有保留原文不动、只在表里标出待用户定?没有擅自改进正文吧?
- 修正稿是不是干净的——没有把来源标注、旧值、"原来错在哪"掺进正文?
- 数量级过没过常识关?有没有"整齐得不像真的"或明显超出常识基准的数混进来?

## 触发短语

- "核实/核查这篇(文稿/文章/报告)的数据"
- "文里的数字/比例/排名/日期对不对""帮我校对数据"
- "更新这篇的数据""数据是不是过期了""把数据刷成最新的"
- "做一下事实核查""fact-check 这段"
- 任何"把已有文稿里的事实性数据查证 + 更新"的诉求

## 底线

文稿里一个没核实过的数字,就是一个潜在的错。这个 skill 的职责是:把每个数据查到底、标清楚状态、改干净能确认的、标出拿不准的。查不到就老实说查不到,绝不留白、绝不凭记忆补。
