# 去 AI 味（中文）

把一段中文改得不像 AI 写的、不像机翻的，像一个把事情想清楚了的人写的。这套方法管两件事：识别 AI 中文写作的典型痕迹，和把它改成落地的写法。

核心是一句话：**AI 味不在某几个词，在于句子搭了架子却没让读者多知道什么。** 先抓这个，再去抠具体的词。

## 怎么用

给定一段草稿：

1. 先用下面的**三个诊断问题**逐句过一遍，定位可疑的句子。
2. 对照**规则**逐条改写，改的时候就避开 AI 味，不是写完再改。
3. 跑一遍**收尾自检**，确认没漏。
4. 输出落地版。拿不准的句子,回 `references/案例库.md` 对照真实改稿。

如果是你自己在生成文字（不是改别人的），同样在生成时就守这套规则，别先写出 AI 味再回头修。

## 先看三件事

AI 味的根子是三件事。一是句子只搭了个架子，读完没多知道什么。二是该讲清机制的地方，拿一个比喻或一句黑话糊弄过去。三是结论本该放前面，却先铺垫、先卖关子，把它藏到后面。

对每一句可疑的话，问自己三个问题：

- **这句让读者多知道了一件具体的事吗？** 只搭架子、读完什么也没多知道的，删掉或者补全。比如"四个理由，三个事后兑现，原因是估值太高"，听着像说了什么，其实读者什么都没多知道——要么当场把四个理由列出来，要么整句删掉直接上结论。
- **这句是在讲谁导致了谁，还是只贴了个标签？** "估值消化是唯一致命变量"像有洞见，其实没说清谁导致了谁；改成"估值的下跌幅度超过了利润的增长幅度"，因果和方向就清楚了。比喻和黑话基本都属于贴标签。
- **这句好看，是因为内容，还是因为句式？** 排比、对仗、三项并列、破折号抖包袱，这些是节奏好听，不是信息变多。把句式拉平，看意思有没有损失；没损失，那点好看就是空的，拉平。

下面每条规则，都是这三个问题落到具体写法上。

## 规则

### 删填充和套话

开头不要"值得注意的是""总的来说""综上所述""不难看出"。连接处不要逢句就"此外""然而""与此同时""更重要的是"。这些词删掉，句子照样成立，而且更干净。

- 删前：值得注意的是，公司的研发投入持续加大，此外，其市场份额也在不断提升。
- 删后：公司研发投入持续加大，市场份额同步提升。

### 不靠否定定义范围，直接写覆盖了什么

不要用"本文不讨论 / 不覆盖 / 不展开 X"这种对文章范围的否定式铺垫来开场，也不要写"这篇做的不是……而是……"的自指声明。读者要的是你做了什么，不是你没做什么。"与其说是……不如说是……"这类否定式拔高也归此类。

- 删前：这篇解读不讨论 AI 是不是泡沫，而是把它变成一道能算的数学题。
- 删后：把 AI 的 Token 经济学变成一道能算的数学题——每 GW 算力能赚多少、成本多少。

**边界**：这条只管"对文章范围 / 内容的否定式铺垫"。正常的分析性否定是真观点，照常用——"我不认为短期会反转""市场担心 X，但我判断 Y""若需求不及预期"这些不在禁止之列，该写就写。

### 避免 -ing 式的伪分析

不要在句尾挂"……，彰显了其行业地位""……，体现了战略眼光""……，为未来增长奠定基础"这类空转的总结。它们看着像分析，其实没给任何新信息。

- 删前：公司推出新一代芯片，彰显了其在国产替代浪潮中的领军地位，为长期发展注入了强劲动力。
- 删后：公司新一代芯片 FP16 算力达 2048 PFLOPS，对标国际主流产品。

### 不堆三段排比

不要凡事凑成三项："创新、协同与突破""技术、市场与生态"。该两项就两项，该四项就四项，该一句话说完就一句话。机械的三段式是 AI 最明显的指纹之一。

### 不滥用破折号和否定排比，结论放前面

少用破折号制造"停顿—揭晓"的效果。也不要用"不仅仅是……更是……""与其说是……不如说是……"这种否定式排比来拔高，直接陈述就行。

- 删前：这不仅仅是一次产品迭代——更是公司战略的全面升级。
- 删后：这次迭代把架构升级到第五代，覆盖 FP32 到 INT4 全精度计算。

结论放前面，别卖关子。"最反直觉的是""最该记住的一条"这种吊胃口的开头，连同用破折号抖包袱的写法，都删掉，先把结论给出来。

### 变化句子节奏

不要让每句都一样长。短句给结论，长句铺理由，混着用。连着三句结构雷同就改一句。读起来像人写的，不像模板填出来的。

### 不写金句

如果一句话读着像可以裱起来的标语（"科技向善，连接未来"），它大概率是空的。要的是能验证的判断，不是口号。删掉重写成具体的。

### 标题用自然中文，带判断

标题按内容起名，不要用翻译腔的抽象词。带判断、带画面的标题比空泛的强：叫"从即时通讯工具到社交巨头""国产替代空间广阔"，不叫"业务概述""市场分析"。

**带判断这条管到所有层级，小标题也一样。** 不要让小节标题停在中性名词短语：

- 改前：CAPEX 总量与节奏
- 改后：CAPEX 高度集中于四家超大厂

读者扫一眼标题，应该就能知道这节要论证什么，而不只是"这节讲 CAPEX"。

### 不写硬翻译词和装饰性比喻

有一类 AI 味比较隐蔽：从英文写作里逐字直译过来的"翻译腔黑话"，和为了"有画面感"硬塞的比喻。用四条判别法识别（能管住词表之外的新词）：

- **直译 calque**：一个词是英文术语逐字直译、中文里没人这么用 → 硬翻译，换成习惯表达。
- **装饰性比喻**：一个比喻删掉后信息不减、只是少了点"画面感" → 装饰性的，删。
- **能留的比喻**：一个比喻能把复杂关系讲得更短更清楚、且本身是通用行话（护城河、卡脖子、量价齐升）→ 可以留，但一段最多一个，不堆砌。
- **生造缩写 / 口语**：一个缩写或口语不是通用书面表达（结构毛、够不着）→ 写全、换成书面说法。

种子词表，三类，每条「AI 味写法 → 改成什么（为什么）」。这是代表不是全集，照判别法举一反三：

**① 英文直译 calque**
- 成本桥 / 收入桥 → 成本构成拆解 / 收入增量拆解（cost/revenue bridge 的直译）
- 顺风 / 逆风 → 利好因素 / 不利因素、顺周期 / 逆周期（tailwind、headwind）
- 价值洼地 → 低估
- 北极星指标 → 核心指标
- 敞口 → 风险暴露、受……影响的规模、涉及金额（exposure 的直译，写清是什么的暴露、多大规模）
- 硬约束 / 软约束 → 硬性限制 / 上限、绕不开的限制 / 弹性空间（"电力是唯一稳定的硬约束"→"电力容量是算力的上限"）
- 飞轮（空喊时）→ 说清是哪个"投入—产出—再投入"的正反馈，别只甩一个"飞轮"

**② 装饰性比喻**
- 上船的船票价 / 这张船票 → 入场门槛、资本开支门槛（直接说门槛）
- 埋下隐忧 / 埋下伏笔 → 带来……风险
- 弯道超车 → 写清靠什么路径超
- 压舱石 → 基本盘 / 稳定来源

**③ 生造缩写 / 口语**
- 结构毛 → 结构性毛利率变化
- 够不着 → 难以承担 / 达不到门槛
- 玩得起 / 玩家 → 有能力投入的公司 / 参与者
- 跑出来 → 胜出 / 跑赢

**可以留的行话**（别误删）：护城河、对标、卡脖子、量价齐升、戴维斯双击——这些是中文里的正经行话，用得着就用，只是别滥用堆砌。注意线划得严：敞口、硬约束虽然常见，但本质是 exposure、hard constraint 的直译，一律换成中文说法，不算可留行话。

### 情绪化和暴力的词换成字面

有一类带情绪、带画面但没信息的词：暴雷、杀估值、吃掉利润、蛋糕在缩、崩盘、破产、救不了、清零。它们听着有劲，其实把该讲清楚的机制盖住了。换成字面的说法——"暴雷"写成"因会计差错更正大幅下滑"，"单边杀估值吃掉利润增长"写成"估值下跌幅度超过利润增长"，"蛋糕在缩"写成"行业总量在萎缩"。更多对照见 `references/案例库.md`。

### 少用对冲词

可能、或许、也许、某种程度上、被认为这类词，一句话最多留一个。想说不确定，用一个明确的词就够，别叠成"该政策或许在某种程度上可能被认为产生某些影响"。

但要分清楚：数据估算时的"约、左右、近、超过、不足"该用就用——"约 7366 亿""近 74%"是老老实实标注估算精度，不是含糊其辞。对观点心虚才删，对数据标精度要留。

### 同一个东西从头到尾用同一个词

一个对象、一个指标、一家公司，认准一个名字用到底，别为了不重复换着叫。把"主营业务"一会儿写成"核心板块"、一会儿写成"主业"、一会儿写成"基本盘"，读者还得猜是不是同一个东西。AI 爱用同义词替换，写的时候别跟着换。

### 表述精确

口语化和模糊因果是另一种不严谨。要书面、要把因果讲明白。

- 口语化程度词（"不小的压力""够不着""挺好的"）→ 换成具体量级或书面表达。
- 模糊因果（"埋下隐忧""带来影响""未必稳得住"）→ 写清「主体 + 机制 + 方向」。
- 删前：集中本身又埋下隐忧。
- 删后：capex 靠债务融资和厂商间相互采购支撑、高度集中于少数几家，资金链一旦收紧会引发财务风险。

### 逻辑链：判断后追一层归因

抛出一个判断或风险，不能停在现象，要再往下一层——讲清**机制**（通过什么传导）和**落点**（落在谁身上、由谁承担）。只说"有风险"不说风险从哪来、压在哪，是 AI 写作最常见的逻辑断点。

- 弱：资金链一旦收紧，投入未必稳得住。
- 强：capex 越来越依赖债务融资和厂商间相互采购，需求高度集中。一旦融资成本上行、或某一家收缩资本开支，采购链上的需求会连锁回落。风险不在单家的资产负债表，而在几家相互绑定的现金流上。

强的版本讲清了机制（债务 + 相互采购传导）和落点（相互绑定的现金流，而非单家）。归因落在哪一环，要明确，不能含糊带过。

## 边界：不要过度去味

去 AI 味是去掉空的东西，不是把内容也削掉。下面这些照留：

- **数字和事实**——一个不改。去味只动措辞，不动数据、不动事实判断。
- **专业行话**——护城河、对标、卡脖子、量价齐升这类通用行话该用就用。
- **分析性否定**——"我不认为""市场担心 X 但我判断 Y"是真观点，不是否定式铺垫。
- **数据精度对冲**——"约、近、超过、左右"标的是估算精度，是严谨，不是心虚。

判别就一句：对观点心虚、靠句式撑场面的，删；对事实负责、把机制讲清的，留。

## 收尾自检

写完一段或一节，回头核对一遍：

- 连着三句一样长？打断一句
- 有"此外""然而""值得注意的是"？大概率能删
- 有"彰显""体现""奠定基础"式的空转尾巴？删掉或换成具体内容
- 凑了三段排比？按内容该几项就几项
- 有读着像口号的金句？重写成可验证的判断
- 判断后面有没有跟具体事实 / 数字？没有就补
- 有没有"本文不讨论 / 不覆盖 X"式的否定铺垫？改成正面写覆盖了什么（分析性否定不算）
- 有没有多余的引言段？删掉
- 有硬翻译词 / 直译 calque（成本桥、顺风逆风、敞口、硬约束）？换成中文说法，行话（护城河、对标）留着
- 有装饰性比喻（删掉信息不减的，船票、埋下隐忧）？删
- 该并列枚举的有没有用 bullet / 分段？该连贯论证的有没有被拆碎？
- 抛出的判断 / 风险，有没有追一层归因（讲清机制 + 落点）？停在现象就补
- 小标题带观点了吗？中性名词短语改成带判断的
- 有情绪化、暴力的词（暴雷、杀估值、吃掉、蛋糕在缩）？换成字面的说法
- 对冲词堆了吗（可能、或许、某种程度上）？一句留一个，数据的"约、近"留着
- 同一个东西有没有换着叫？认准一个词用到底
- 数字和事实有没有被误删或改动？去味只动措辞

最后一道总检查：把这段话里的形容词、比喻、对仗都拿掉，看还剩多少具体信息。剩得越多越好；要是拿掉就没剩什么，说明本来就在说空话，重写。拿不准的句子，回去对照 `references/案例库.md`。
