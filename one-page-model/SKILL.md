# One Page Model（一页式损益模型）

为任意一家公司建立一页式损益模型，回答两个核心问题：未来五年每年净利润是多少；成本花在哪里、各花多少。交付物是一个所有假设可单独替换、所有数字可点回来源的 Excel 活模型。

适用公司分三级，数据策略随级别变，模型框架不变：

| 级别 | 界定 | 主数据源 | 占比 |
|------|------|----------|------|
| A 上市公司 | 在任一交易所挂牌，有审计财报 | 10-K/10-Q/年报/中报、业绩指引、分析师一致预期 | 绝大多数任务 |
| B 准上市公司 | 已递交 S-1/招股书，或处于 IPO 进程 | S-1/招股书审计数 + 招股条款 | 常见 |
| C 私有公司 | 无任何审计披露 | 监管文件中的间接锚 + 一线媒体转引 + 数据商 | 少数，难度最高 |

判定动作放在 Phase 0：先确认公司属于哪一级，再决定检索什么。判错级别整个数据策略都会错。

这个模型给的是数量级判断和盈亏平衡路径。别指望它精确到小数点，那是伪精度。

**Key Characteristics:**
- **预测期**: 基准年 + 向后 5 年
- **情景**: 基准 / 乐观 / 悲观 三档，每档独立假设、独立转正年份
- **单位**: 十亿（B）或亿，保留一位小数，全表统一
- **公式**: 模型页 100% 活公式，硬编码输入只存在于假设表
- **数据**: 每个进模型的数字必须来自当次实际检索、带日期的来源
- **自检**: 模型回算结果与外部独立锚点偏差 ≤30%，否则回头改假设
- **交付**: 单个 .xlsx 文件，固定 Sheet 顺序与配色规范

## When to Use

用户提出以下类型请求时使用：
- "给 [公司] 建一个财务模型 / 损益模型 / 盈利预测"
- "测算 [公司] 什么时候盈亏平衡 / 何时转正"
- "[公司] 未来五年净利润 / 成本结构是什么样"
- "参考 [已有模型] 对 [另一家公司] 做一个同样的模型"

**Do NOT use if:**
- 用户要上市公司的精细三表联动模型 → 超出"简易"定位，需另行约定颗粒度
- 用户要 DCF / 可比公司估值 → 估值是另一类任务，本模型只产出损益
- 用户只是问某个财务数字 → 直接检索回答，不必建模

## Critical Requirements

### 1. 禁用训练记忆（最高优先级）

训练数据是过时的。进入模型的每一个数字，必须来自本次实际检索到的、带日期的来源。
- 开始前写下今天的日期，所有"最新"数据以此为基准核验
- 无法联网时立即终止任务并说明原因，不得产出任何数字
- 用户给的数字也要检索交叉验证。用户的数字可能同样过时，实践中出现过基准收入因为新披露要上调 50% 的情况
- 上市公司同样适用：最近一季财报、最新指引、重大并购都可能在训练截止后发生

### 2. 五字段数据底稿

每个数据点记五个字段：数值、口径、时间点、来源（含链接）、置信度。缺一个都不行。
找不到的数据标注「未找到」，用显式假设替代并写明依据，禁止编造。

### 3. 口径四分法

年化 run-rate、日历年确认收入、合同总额、现金消耗，是四种性质不同的数字，不能互相替代。
私有公司的数据错误，大半出在这里。详见 [references/workflow.md](references/workflow.md) 第一阶段。

### 4. 硬锚点优先

锚点按公司级别取：
- A 级（上市）：公司自己的审计财报就是锚，分部数据、指引、一致预期直接可用，不需要绕路
- B 级（准上市）：S-1/招股书审计数为锚，历史年度可回测 
- C 级（私有）：最可信的数字往往不在它自己嘴里。按可信度排序去三类地方找：关联上市公司的 10-K/10-Q（权益法损益可反推净亏损、关联交易、采购承诺）；宣誓证词与法庭文件；一线媒体（The Information、WSJ、Bloomberg、Reuters、FT 级别）转引的内部预测

### 5. 假设与事实严格分开

全部假设集中进假设表，每条写明数值、依据、所属情景。任何一条假设都能单独替换，不动数据底稿。模型页只引用假设表，不许有第二处硬编码。

### 6. 自检回路（不可跳过）

自检方式按数据级别定：
- A/B 级（有审计实际数）：回测是标配。用模型逻辑还原最近 1-2 个审计年度，分项偏差 <10% 才允许外推；A 级另需对照分析师一致预期与公司指引（基年与次年），偏离要能说出理由
- C 级（无审计数）：模型基准年与次年净亏损对外部锚点（披露亏损、现金消耗、权益法反推），相对偏差 ≤30% 或绝对偏差 ≤基年收入 5% 为通过；不通过回头改假设并记录

## High-Level Workflow

完整流程分五个阶段，详细操作见 [references/workflow.md](references/workflow.md)：

### Phase 1: 数据采集（联网检索）
先判定公司级别（Phase 0），再按级别采集四类数据：收入、成本与盈亏、硬锚点、校验参照（估值/融资/市值，只验量级不进损益）。来源分级、双源印证、冲突记录。

### Phase 2: 量级三道关
每个数字进模型前过三关：常识基准对比、单位统一、口径性质区分。任何一关不过即标可疑，重新核验。

### Phase 3: 建模
先做结构决策：收入建单线还是分部各建一条，成本按科目拆还是用分部利润率倒推，取决于披露口径长什么样。然后设三档情景，最后跑自检或回测。

### Phase 4: Excel 工程
按固定的 Sheet 顺序、配色规范、公式架构落成文件，重算校验做到零错误。详见 [references/model-structure.md](references/model-structure.md)。

### Phase 5: 结论与质检
输出对比结论（转正年份矩阵、最敏感的 2-3 条假设）与局限性声明，过质量检查清单。详见 [references/best-practices.md](references/best-practices.md)。

## Output Specification

**Primary Deliverable**: 单个 .xlsx 文件
**File Name**: `[公司名]_OnePageModel.xlsx`（多公司对比则 `[公司A]_[公司B]_OnePageModel.xlsx`）

**固定 Sheet 顺序**（用户另有要求时以用户为准）:
1. `[公司]模型`（每家公司一页，左上角大字标题"[公司名]财务推算模型"，三情景块）
2. `数据来源`（五字段底稿）
3. `假设表`（蓝色输入集中地）
4. 支撑页：成本推算 / 分部回测 / 融资历史 / 对比结论 / 局限性 / 说明

**配色规范**（财务建模通行约定）:
- 蓝色字 = 硬编码输入（只允许出现在假设表）
- 黑色字 = 本表公式
- 绿色字 = 跨表引用
- 黄色底 = 关键敏感假设

## Quality Bar

好的输出：每个数字能点回来源，假设能被单独质疑和替换，改假设表里任意一个蓝色数字全表自动重算，读者看完能说出在什么假设下、哪一年、转正与否。

坏的输出：数字无来源，拿估值当利润依据，用精确数字掩盖数据缺失，情景之间只差一个没有依据的百分比，公式里藏硬编码。

## Dependencies

**Required:**
- 联网检索能力（web search + fetch），无法联网必须终止
- Python + openpyxl（建文件），xlsx skill 的 recalc 脚本（公式校验）

**Optional:**
- 监管文件检索（SEC EDGAR、港交所披露易、巨潮资讯等），A/B 级公司和有上市关联方的 C 级公司都建议直查原文件
