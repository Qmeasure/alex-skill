# Excel Model Structure & Engineering Spec

Excel 文件的工程规范，三级公司通用。目标只有一个：用户改假设表里任意一个蓝色数字，全表自动重算，结论页跟着更新。

## Sheet 顺序（默认模板）

| # | Sheet | 内容 | 性质 |
|---|-------|------|------|
| 1..n | `[公司]模型` | 每家公司一页，三情景块 | 全活公式 |
| n+1 | `数据来源` | 五字段底稿 + 口径冲突记录 | 静态 |
| n+2 | `假设表` | 全部蓝色输入 + 每条依据 | 输入层 |
| n+3.. | 支撑页 | 成本推算 / 分部回测 / 融资历史 / 对比结论 / 局限性 / 说明 | 静态+引用 |

用户对顺序、标题另有要求时以用户为准。打开文件默认停在第一个模型页（`wb.active = 0`）。

## 模型页布局

标题行：第 1 行独立大字标题「[公司名]财务推算模型」，16pt 加粗，合并至数据区最右列，行高加大。一页只标一家公司的名字。

**情景块**（基准/乐观/悲观，纵向排列，块间空一行）：

科目拆分法的行结构：
```
情景名（灰底加粗）
（$B）| 2026 | 2027 | 2028 | 2029 | 2030   ← 深蓝底白字表头
收入
收入成本
[最大战略性支出项，如研发]
SG&A
利息及其他（有息负债重大时必加）
税项（转正后年份，或全模型声明税前口径）
净利润（加粗）
净利率
净利润转正年份（公式判定，见下）
```

分部利润率法的行结构：
```
[分部1]收入 / [分部2]收入 / [分部3]收入 / 总收入
[分部1]营业损益 / [分部2]营业损益 / [分部3]营业损益 / 合并营业利润
利息及其他
净利润（加粗）/ 净利率 / 转正年份
```

## 公式架构

数据只有一个流向：假设表（蓝色输入）→ 模型页（引用+计算）→ 结论页（引用模型页）。

核心公式模式（B 列为基年）：
```
收入基年:    =假设表!B{基年收入行}
收入次年起:  =上年单元格*(1+假设表!{本列}{增速行})
成本(率法):  =收入单元格*(1-假设表!{本列}{毛利率行})        # 或 *利润率
成本(绝对额): =假设表!{本列}{支出行}
SG&A:        =收入单元格*假设表!$B${比例行}
净利润:      =收入-各成本块                                 # 或 Σ分部+利息
净利率:      =净利润/收入
转正年份:    =IF(B{ni}>0,B${表头行}&"",IF(C{ni}>0,C${表头行}&"",...,"五年内不转正"))
             # 年份取自表头单元格而非硬编码字符串，基年变更时自动跟随
净利率防护:  =IF(收入=0,"-",净利润/收入)
```

结论页的转正矩阵和净利润汇总行全部用 `=[公司]模型!{单元格}` 跨表引用（绿色字），不抄数字。

## 配色与格式规范

| 元素 | 规范 |
|------|------|
| 字体 | Arial 全表统一 |
| 蓝色字 | 硬编码输入，**只允许出现在假设表** |
| 黑色字 | 本表公式 |
| 绿色字 | 跨表引用 |
| 黄色底 | 关键敏感假设（每家公司标 1-2 条） |
| 数字格式 | `$#,##0.0;($#,##0.0);"-"`（负数括号、零横杠） |
| 百分比 | `0.0%;(0.0%);"-"` |
| 表头 | 深蓝底（1F4E79）白字加粗居中 |
| 长文本 | wrap_text + 调行高，列宽按内容设置 |

## 假设表布局

每个情景一个块：`假设项 | Y1 | Y2 | Y3 | Y4 | Y5 | 依据`。

- 基年值只填 B 列；增速从 C 列起；率类逐年填
- 依据列必须写：引用底稿编号，加一句话逻辑，格式如"锚定 [底稿编号]：[披露内容]，[档位] 取 [处理方式]"
- 关键敏感假设在依据列加【关键敏感假设】标记并黄色高亮数值

## openpyxl 实现要点

1. 先读 xlsx skill 的 SKILL.md，按它的环境规范操作
2. 用构建脚本生成（`build_[公司].py`），不要在已有文件上手工逐格改。脚本即版本记录，迭代时改脚本重建，避免行号漂移弄断引用
3. 行号管理是最大的坑。块结构一改（比如插入标题行），所有跨表引用的行号都要同步修正。在脚本里把"假设表各块数据首行"作为参数传给模型页构建函数，别散落硬编码
4. 合并单元格用于标题；`wb._sheets` 重排顺序；改 Sheet 名后跨表公式由 openpyxl 自动跟随，但仍需重算验证
5. 必跑校验：
   ```
   python <recalc脚本> 文件.xlsx 60     # 公式重算，要求 total_errors=0
   ```
   然后 `load_workbook(data_only=True)` 读回关键输出（每情景的净利润行、转正年份），与手算预期逐项核对一致后才交付
6. 每次迭代交付前重复第 5 步，把文件复制到输出目录，用文件展示工具呈现

## 支撑页模板

成本推算页（最大成本项有自下而上逻辑时）：
- 推算链条表：历史实际 → 当年锚 → 远期计划，标注每个数的锚点性质
- 交叉验证记录，比如分项之和等于独立披露的总额，链条闭合
- 量级三道关的检查记录
- 可持续外推逻辑：支出 ≈ 驱动变量 × 单位成本，驱动变量延期则锚同步下调

回测页（有审计实际数时）：
- `项目 | 模型逻辑回算 | 审计实际 | 偏差 | 说明` 五列表
- 回测结论 + 前瞻自检的差异解释

融资历史页（B/C 级用；A 级换成资本结构页，列有息负债、到期结构、回购分红，用途同样是校验资金可持续性）：
- 每轮：时间 | 轮次 | 金额 | 投后估值 | 关键投资人 | 备注 | 置信度
- 未确认的轮次明确标注，金额归入数据商汇总口径的差额，不编数
- 表末固定一条「与模型的对照」：累计融资对模型情景的累计亏损，判断资金链压力

说明页：建模日期与版本、单位、配色规范、口径声明、自检记录。迭代时版本号递增，写明本版变更。
