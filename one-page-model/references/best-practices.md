# Best Practices, Examples & Quality Checklist

## 好结论 vs 坏结论

### 好的结论表述（可检验、有条件、落在假设上）：
- "基准情景下 [公司A] 于 Y3 转正，[公司B] 五年内不转正，第五年末约 -X。分化的最大单一驱动是 [成本项] 的 N 倍差距"
- "即使乐观情景（= 公司自己的收入计划全额达成），转正也要到 Y5，说明其盈亏开关完全押在收入端，成本端已被锚死"
- "砍掉 [利润率改善] 这条假设，Y2 转正立即消失"
- "累计融资 $X vs 基准累计亏损 $Y：按基准路径现有资金到 Y5 基本耗尽，必须持续融资"

### 坏的结论表述（避免）：
- "公司前景广阔、盈利能力强劲"。无条件，不可检验
- "预计 2029 年净利润为 50.3 亿美元"。伪精度，应表述为量级与区间
- "估值已达 $X，说明盈利可期"。拿估值当利润依据
- 只给一档结果，不给情景区间

## 迭代更新协议（用户质疑数据时）

用户说某数据有误、过时、需要再验，这是常态。按此处理：

1. 先验真。针对被质疑的数据块做 1-3 次专项检索，确认用户对不对。可能用户对，披露更新了；也可能是口径差异，比如 run-rate 对确认收入
2. 找到根因再改。明确旧数错在哪：过时、口径、还是单源失真，新锚是什么等级的来源
3. 连锁更新：改假设表对应数值 → 同步更新数据底稿条目，旧条目改写而非删除，保留口径冲突记录 → 更新依据列文字 → 更新自检记录 → 重建重算
4. 报告结论位移。明确告诉用户哪些结论变了、哪些没变、为什么。"转正年份不变但幅度下调"也是重要信息
5. 版本递增，说明页记录每版的变更内容与日期

一条纪律：被质疑后的修正可能是双向的。一家公司数据上调时，另一家可能要下调，不要默认更新就是调高。

## 常见错误清单

1. 用训练记忆里的数字。头号错误，高增长公司的收入按月过时
2. run-rate 当确认收入。高增长期会系统性高估基年
3. 合同总额当年度费用。多年采购承诺除以年限，只能作量级参照
4. 现金消耗当会计亏损，或者反过来。两者差距可达数倍，口径必须声明
5. 多家媒体转引同一篇报道当"双源"。追到信源头再判定独立性
6. 内部预测锚不看前提。公司内部的亏损预测可能基于已过时的收入预测，直接当锚会冤枉或美化现状，要拆出差异来自收入端还是成本端
7. 悲观情景只砍收入不砍支出。资金约束下支出必然收缩，不砍会把悲观亏损算大
8. 乐观情景凭空加成。乐观档应锚定公司自己的公开计划，不是拍一个更高的数
9. 公式里藏硬编码。藏一处，整个"可替换假设"的承诺作废
10. 改结构不改引用。插入标题行、重排 Sheet 之后跨表引用行号会断，必须重算校验加读回核对
11. 付费口径、调整后口径与整体口径混用。"付费用户利润率"不等于整体毛利率，单列并标注
12. 拿估值或融资额进损益。校验参照永远不进计算
13. 税前当净利润。转正后年份不计税会系统性高估 20-25%，要么粗算税，要么明示税前口径
14. 漏掉利息。重债公司的利息可达单年亏损量级，科目法尤其容易漏
15. 盈亏平衡点附近迷信相对偏差。亏损基数趋零时 30% 的相对偏差没有约束力，改用绝对偏差或分项锚
16. 非美元公司不声明汇率口径。折算汇率与时点写进说明页
17. 把上市公司当私有公司建模。放着审计财报和一致预期不用，去拼二手报道，数据质量自降两档
18. GAAP 与 non-GAAP 混用。上市公司头号口径陷阱，调整项（SBC、摊销、一次性）要么全剔要么全留，并声明
19. 财年与日历年错位。财年截止非 12 月的公司，所有同比和基年定义先统一口径再录入
20. 一次性项当经常性。减值、重组、诉讼和解混进成本基数，会把利润率路径整体带偏
21. 无视公司指引与一致预期。A 级公司的基准档凭空另起炉灶，偏离锚而不写理由

## 交付前质量检查清单

**数据层：**
- [ ] 公司级别（A 上市/B 准上市/C 私有）已判定且写入说明页
- [ ] 每个进模型的数字都有五字段底稿条目
- [ ] 关键数据双源印证，单源已标注
- [ ] 口径冲突已记录并写明采信理由
- [ ] 所有数字过了量级三道关
- [ ] 今天的日期已写入说明页，时效敏感数据均为近期检索

**模型层：**
- [ ] 蓝色输入只存在于假设表，每条有依据
- [ ] 三档情景差异均可溯源，增速逐年衰减
- [ ] 自检/回测已执行且通过（或差异已解释并记录）；A 级另已对照指引与一致预期
- [ ] 转正年份由公式判定且年份引用表头，非手填
- [ ] 税项与利息的处理方式已声明（计算 or 口径声明）
- [ ] 改任意假设可重算（抽一条试改验证后还原）

**工程层：**
- [ ] recalc 零错误，公式总数符合预期
- [ ] data_only 读回的关键输出与手算预期一致
- [ ] Sheet 顺序、标题、配色符合规范（或用户的明确要求）
- [ ] 文件已放输出目录并呈现给用户

**结论层：**
- [ ] 转正年份矩阵 + 最敏感 2-3 条假设已写明
- [ ] 与公司公开口径的互洽性已检查
- [ ] 局限性页含"输出是数量级区间"声明
- [ ] 回复用户时报告了本版相对上版的结论位移
