# 研报写作规范

研报里的每一段文字都按这份规范写。它管两件事：一是中文卖方研报该怎么写，二是怎么不写出 AI 味和翻译腔。写之前通读一遍，写完再回头核对一遍。

## 一、研报怎么写

### 投资要点：加粗领起 + 展开

报告摘要和每章开头的要点，都用"一句加粗的判断 + 几句展开"的结构。加粗句给结论，后面的句子给理由和数据。

看标杆怎么写：

> **C 端基业长青，B 端蓄势待发。** 在不考虑新流量入口的前提下，MAU 预计基本稳定；增值服务形成闭环护城河，ARPU 伴随经济增长稳步提升；营销服务的量价空间较大，云服务门槛进一步降低后存在渗透率和集中度的两重机会。

加粗的是判断（"基业长青、蓄势待发"），后面把判断拆成 MAU、ARPU、营销、云四条具体理由。不要反过来——先罗列大量描述，最后才得出结论。

### 数字优先

研报靠数字说话，不靠形容词。

- 不写"营收大幅增长"，写"营收同比增长 65.6% 至 11.74 亿元"
- 不写"盈利能力显著改善"，写"亏损收窄至 -4.52 亿元，2025 前三季归母净利 16.05 亿元，同比增长 321.49%"
- 不写"市场空间广阔"，写"IDC 预测 2029 年中国 AI 芯片市场规模达 13367.92 亿元，2025–2029 年均复合增长 53.7%"

每个判断后面尽量跟一个数。拿不到精确数就给区间或量级，但要标清楚这是估算。

### 用研报的动词

中文研报有一套固定的分析动作，用这些词，读者一看就知道你在做什么分析：

- **复盘**——回看历史（历史复盘、股价复盘）
- **拆分**——把总量拆成结构（分部拆分、量价拆分、收入拆分）
- **测算**——基于假设算出结果（盈利测算、空间测算、弹性测算）
- **对标**——找参照（对标海外龙头、对标一致预期）
- **演绎 / 推演**——顺着逻辑往下推（情景推演）

比如"营收按业务分部拆分后，游戏贡献测算约占四成"比"我们分析了营收构成"要有研报味得多。

### 年份口径：A 和 E

历史年份标 A（实际，actual），预测年份标 E（estimate）。盈利预测及估值表里一定要分清，比如 2023A / 2024A / 2025E / 2026E / 2027E。这是研报的硬惯例，读者靠这个一眼分清哪些是已发生、哪些是预测。

### 来源标注

每张图、每张表、每个关键数据都标来源，格式统一：

> 来源：公司公告、WIND、XX证券研究所

公司自己披露的标"公司公告""公司年报"；第三方数据标具体数据商或机构（WIND、IDC、Counterpoint）；自己测算的标"XX研究所测算"或"XX研究所预测"。多个来源用顿号连。不要写笼统的"公司数据""网络资料"。

### 加粗只给结论

加粗留给真正的判断和关键数据，不要满篇加粗。一段里加粗一两处，读者的视线才知道往哪看。把每个名词都加粗，等于没加粗。

### 枚举用并列，推演用散文

内容的形态决定排版。判别线很简单：

- **并列枚举**（N 类玩家、N 个驱动、N 条风险这种平行罗列）→ 用 bullet 或分段，每项一句加粗判断领起，再展开。
- **连续推演**（一环扣一环的因果、单一论点的论证）→ 用段落散文，别拆碎。

两头都要防：该并列的塞成一段流水账，读者拎不清有几类；该连贯论证的拆成一堆短 bullet，研报就成了 PPT。

- 反例（两类玩家塞一段）：能玩得起的只有两类，一类是 capex 能转产能的云厂商，另一类是盈利模式极强的广告巨头……
- 正例（拆成两个 bullet，各加粗领起）：
  - **能快速回流 capex 的云厂商。** 算力建好就有客户租，投入快速转成收入。
  - **现金流养得起 AI 的广告巨头。** 谷歌、Meta、字节用主业现金流支撑资本开支，不依赖外部融资。

## 二、怎么不写出 AI 味

下面这些是 AI 中文写作的典型痕迹，研报里出现一次就掉一档专业度。生成时就避开，不是写完再改。

### 先看这三件事

AI 味的根子不在某几个词，主要是三件事：句子只搭了个架子，没让读者多知道什么；该把机制讲清楚的地方，用一个比喻或一句黑话糊弄过去；该先给结论的地方，先铺垫、先卖关子，把结论藏在后面。先盯住这三件事，再去抠下面那些具体的词。

写的时候，对每一句可疑的话，问自己三个问题：

- **这句让读者多知道了一件具体的事吗？** 只搭架子、读完什么也没多知道的，删掉或者补全。比如摘要里"四个理由，三个事后兑现，原因是估值太高"，听着像说了什么，其实读者什么都没多知道——要么当场把四个理由列出来，要么整句删掉直接上结论。
- **这句是在讲谁导致了谁，还是只贴了个标签？** "估值消化是唯一致命变量"像有洞见，其实没说清谁导致了谁；改成"估值的下跌幅度超过了利润的增长幅度"，因果和方向就清楚了。比喻和黑话基本都属于贴标签。
- **这句好看，是因为内容，还是因为句式？** 排比、对仗、三项并列、破折号抖包袱，这些是节奏好听，不是信息变多。把句式拉平，看意思有没有损失；没损失，那点好看就是空的，拉平。

后面每条规则，都是这三个问题落到具体写法上。详细的改写例子在 `去AI味案例库.md`，拿不准的句子回去对照着查。

### 删填充和套话

开头不要"值得注意的是""总的来说""综上所述""不难看出"。连接处不要逢句就"此外""然而""与此同时""更重要的是"。这些词删掉，句子照样成立，而且更干净。

- 删前：值得注意的是，公司的研发投入持续加大，此外，其市场份额也在不断提升。
- 删后：公司研发投入持续加大，市场份额同步提升。

### 不靠否定定义范围，直接写覆盖了什么

不要用"本文不讨论 / 不覆盖 / 不展开 X"这种对文章范围的否定式铺垫来开场，也不要写"这篇解读做的不是……而是……"的自指声明。读者要的是你做了什么，不是你没做什么。直接正面写覆盖了什么。"与其说是……不如说是……"这类否定式拔高也归此类。

- 删前：这篇解读不讨论 AI 是不是泡沫，而是把它变成一道能算的数学题。
- 删后：把 AI Token 经济学变成一道能算的数学题——每 GW 算力能赚多少、成本多少。

**边界**：这条只管"对文章范围 / 内容的否定式铺垫"。研报里正常的分析性否定是真观点，照常用——"我们不认为短期会反转""市场担心 X，但我们判断 Y""若需求不及预期"这些不在禁止之列，该写就写。

### 避免 -ing 式的伪分析

不要在句尾挂"……，彰显了其行业地位""……，体现了公司的战略眼光""……，为未来增长奠定基础"这类空转的总结。它们看着像分析，其实没给任何新信息。

- 删前：公司推出新一代芯片，彰显了其在国产替代浪潮中的领军地位，为长期发展注入了强劲动力。
- 删后：公司新一代芯片 FP16 算力达 2048 PFLOPS，对标国际主流产品。

### 不堆三段排比

不要凡事凑成三项："创新、协同与突破""技术、市场与生态"。该两项就两项，该四项就四项，该一句话说完就一句话。机械的三段式是 AI 最明显的指纹之一。

### 不滥用破折号和否定排比

少用破折号制造"停顿—揭晓"的效果。也不要用"不仅仅是……更是……""与其说是……不如说是……"这种否定式排比来拔高，直接陈述就行。

- 删前：这不仅仅是一次产品迭代——更是公司战略的全面升级。
- 删后：这次迭代把架构升级到第五代，FP32 到 INT4 全精度计算。

结论放前面，别卖关子。"最反直觉的是""最该记住的一条"这种吊胃口的开头，还有用破折号抖包袱的写法，都删掉，先把结论给出来。

### 变化句子节奏

不要让每句都一样长。短句给结论，长句铺理由，混着用。连着三句结构雷同就改一句。读起来像人写的，不像模板填出来的。

### 不写金句

如果一句话读着像可以裱起来的标语（"科技向善，连接未来"），它大概率是空的。研报要的是能验证的判断，不是口号。删掉重写成具体的。

### 章节标题用自然中文

标题按内容起名，不要用翻译腔的抽象词。这份 Skill 自己就示范了——叫"执行标准""进入正式产出前""不走捷径"，不叫"执行契约""产物合同读取与输出门槛"。研报章节也一样：叫"历史复盘：从即时通讯工具到社交巨头""GPU 市场需求爆发，国产替代空间广阔"，这种带判断、带画面的标题，比"业务概述""市场分析"这种空泛的标题强。

**带判断这条管到所有层级，小标题也一样。** 不要让小节标题停在中性名词短语，把核心观点放进标题里：

- 改前：CAPEX 总量与节奏
- 改后：CAPEX 高度集中于四家超大厂

一个读者扫一眼标题，应该就能知道这节要论证什么，而不只是"这节讲 CAPEX"。

### 不写硬翻译词和装饰性比喻

还有一类 AI 味比上面更隐蔽：从英文金融/科技写作里逐字直译过来的"翻译腔黑话"，和为了"有画面感"硬塞的比喻。先用三条判别法识别（能管住下面词表之外的新词）：

- **直译 calque**：一个词是英文术语逐字直译、中文卖方研报里没人这么用 → 硬翻译，换成研报的习惯表达。
- **装饰性比喻**：一个比喻删掉后信息不减、只是少了点"画面感" → 装饰性的，删。
- **能留的比喻**：一个比喻能把复杂关系讲得更短更清楚、且本身是行业通用行话（护城河、卡脖子、量价齐升）→ 可以留，但**一段最多一个，不堆砌**。
- **生造缩写/口语**：一个缩写或口语不是研报通用行话（结构毛、够不着）→ 写全、换成书面专业表达。

种子词表，三类，每条「AI 味写法 → 研报怎么写（为什么）」。这是代表，不是全集，照判别法举一反三：

**① 英文直译 calque**

- 成本桥 / 收入桥 → 成本构成拆解 / 收入增量拆解（cost bridge、revenue bridge 的直译，研报说"拆解"）
- 顺风 / 逆风 → 利好因素 / 不利因素、顺周期 / 逆周期（tailwind、headwind）
- 价值洼地 → 低估
- 北极星指标 → 核心指标
- 敞口 → 风险暴露、受……影响的规模、涉及金额（exposure 的直译，写清是什么的暴露、多大规模）
- 硬约束 / 软约束 → 硬性限制 / 上限、绕不开的限制 / 弹性空间（hard/soft constraint 的直译，"电力是唯一稳定的硬约束"→"电力容量是算力的上限"）
- 飞轮（空喊时）→ 说清是哪个"投入—产出—再投入"的正反馈，别只甩一个"飞轮"

**② 装饰性比喻**

- 上船的船票价 / 这张船票 → 入场门槛、资本开支门槛（直接说门槛，案例里"奔向 1.5 万亿是上船的船票价"整句删，只留"每年上千亿的投入门槛"）
- 埋下隐忧 / 埋下伏笔 → 带来……风险
- 弯道超车 → 写清靠什么路径超
- 压舱石 → 基本盘 / 稳定来源

**③ 生造缩写 / 口语**

- 结构毛 → 结构性毛利率变化（生造简写，写全）
- 够不着 → 难以承担 / 达不到门槛
- 玩得起 / 玩家 → 有能力投入的公司 / 参与者
- 跑出来 → 胜出 / 跑赢

**可以留的行话**（别误删）：护城河、对标、卡脖子、量价齐升、戴维斯双击——这些是中文研报的正经行话，用得着就用，只是别滥用堆砌。注意这条线划得严：敞口、硬约束这种虽然常见，但本质是 exposure、hard constraint 的直译，一律换成上面的中文说法，不算可留行话。

### 情绪化和暴力的词换成字面

还有一类带情绪、带画面但没信息的词，研报里常见：暴雷、杀估值、吃掉利润、蛋糕在缩、崩盘、破产、救不了、清零。它们听着有劲，其实把该讲清楚的机制盖住了。换成字面的说法——"暴雷"写成"因会计差错更正大幅下滑"，"单边杀估值吃掉利润增长"写成"估值下跌幅度超过利润增长"，"蛋糕在缩"写成"行业总量在萎缩"。更多对照在 `去AI味案例库.md`。

### 少用对冲词

可能、或许、也许、某种程度上、被认为这类词，一句话最多留一个。想说不确定，用一个明确的词就够了，别叠成"该政策或许在某种程度上可能被认为产生某些影响"。

但要分清楚：数据估算时的"约、左右、近、超过、不足"该用就用——"约 7366 亿""近 74%"是老老实实标注估算精度，不是含糊其辞。对观点心虚才删，对数据标精度要留。

### 同一个东西从头到尾用同一个词

一个分部、一个指标、一家公司，认准一个名字用到底，别为了不重复换着叫。把"主营业务"一会儿写成"核心板块"、一会儿写成"主业"、一会儿写成"基本盘"，读者还得猜是不是同一个东西。AI 爱换同义词，人写研报不换。

### 表述精确

口语化和模糊因果是另一种不严谨。研报要书面、要把因果讲明白。

- 口语化程度词（"不小的压力""够不着""挺好的"）→ 换成具体量级或书面表达。
- 模糊因果（"埋下隐忧""带来影响""未必稳得住"）→ 写清「主体 + 机制 + 方向」。
- 删前：集中本身又埋下隐忧。
- 删后：capex 靠债务融资和厂商间相互采购支撑、高度集中于少数几家，资金链一旦收紧会引发财务风险。

### 逻辑链：判断后追一层归因

抛出一个判断或风险，不能停在现象，要再往下一层——讲清**机制**（通过什么传导）和**落点**（风险落在谁身上、由谁承担）。只说"有风险"不说风险从哪来、压在哪，是 AI 写作最常见的逻辑断点。

- 弱：资金链一旦收紧，投入未必稳得住。
- 强：capex 越来越依赖债务融资和厂商间相互采购，需求高度集中。一旦融资成本上行、或某一家收缩资本开支，采购链上的需求会连锁回落——风险不在单家的资产负债表，而在几家相互绑定、一荣俱荣一损俱损的现金流上。

强的版本讲清了机制（债务 + 相互采购传导）和落点（相互绑定的现金流，而非单家）。归因到银行端还是企业自由现金流端、落在哪一环，都要明确，不能含糊带过。

## 三、收尾自检

写完一段或一节，回头核对一遍：

- 连着三句一样长？打断一句
- 有"此外""然而""值得注意的是"？大概率能删
- 有"彰显""体现""奠定基础"式的空转尾巴？删掉或换成具体数字
- 凑了三段排比？按内容该几项就几项
- 有读着像口号的金句？重写成可验证的判断
- 判断后面有没有跟数字和来源？没有就补
- 加粗是不是太多？只留结论和关键数
- 有没有"本文不讨论 / 不覆盖 X"式的否定铺垫？改成正面写覆盖了什么（分析性否定不算）
- 有没有多余的引言段？删掉，摘要开头那句点题设问已经点过题
- 有硬翻译词 / 直译 calque（成本桥、顺风逆风、敞口、硬约束）？换成研报表达，行话（护城河、对标）留着
- 有装饰性比喻（删掉信息不减的，船票、埋下隐忧）？删
- 该并列枚举的有没有用 bullet / 分段？该连贯论证的有没有被拆碎？
- 抛出的判断 / 风险，有没有追一层归因（讲清机制 + 落点）？停在现象就补
- 小标题带观点了吗？中性名词短语改成带判断的
- 有情绪化、暴力的词（暴雷、杀估值、吃掉、蛋糕在缩）？换成字面的说法
- 对冲词堆了吗（可能、或许、某种程度上）？一句留一个，数据的"约、近"留着
- 同一个东西有没有换着叫？认准一个词用到底

最后一道总检查：把这段话里的形容词、比喻、对仗都拿掉，看还剩多少具体信息。剩得越多越好；要是拿掉就没剩什么，说明本来就在说空话，重写。拿不准的句子，回去对照 `去AI味案例库.md`。
