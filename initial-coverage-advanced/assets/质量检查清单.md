# 质量检查清单

交付前逐项核对。分阶段列，每阶段产物落地时先核对本阶段的，最终报告交付前再核对全表一遍。任一项不达标，补充完善后再交付。

## 交付前阻断项（和渲染级 QA 同级，不过不交付）

这几条是硬门，不是软标准——上一次出事就出在"数据真实只靠自觉"。

- [ ] 跑 `scripts/anti_fabrication_lint.py`（图表构建脚本 + 各 `_data` 底稿 + 稿件）通过，命中已逐条清除
- [ ] 正文和图表里每个数都能在 `_data` 底稿溯源（数值、口径、时间点、来源对得上），没有底稿外的数
- [ ] 没有合成 / 插值 / 示意 / 凑数的数据；缺数的地方是留空或标「未获取」，不是编一个值
- [ ] 现价、市值、指数等时效数据取的是结算日线收盘一手源，不是盘中价或搜索摘要
- [ ] DOCX 渲染成 PDF/PNG 做过视觉核验

## 阶段 1 公司研究稿

- [ ] 按投资逻辑链组织（复盘→业务→财务），不是资料清单堆叠
- [ ] 历史复盘有沿革、关键节点、股价复盘
- [ ] 业务按分部拆分，每个分部的收入占比、变现模式讲清
- [ ] 股权结构、实控人、核心管理层背景到位
- [ ] 行业空间有市场规模数（带来源和年份）、增长驱动、政策
- [ ] 竞争格局覆盖 5–10 家可比，有具体数据不只是罗列
- [ ] 风险 8–12 条，分类
- [ ] 每个关键数字有来源和日期，关键数据双源（且双源异源，非转引同一篇）
- [ ] 建了 `_data` 底稿（五字段），每个进稿数字能溯源；底稿外的数没落进稿子
- [ ] 关键数就近标了口径（招股书 / 年报、绿鞋前后、账面 / 经调整）
- [ ] 全文无 AI 味（对照 `研报写作规范.md` 自检）

## 阶段 2 盈利预测模型（Excel）

- [ ] 蓝色输入只在假设表，每条假设有依据
- [ ] 模型页全部为联动公式，改假设表任一蓝色数字全表重算
- [ ] 收入按分部建线，各分部独立增速驱动
- [ ] 三档情景（乐观/中性/悲观），每档假设可溯源，差异讲得出依据，不是基准 ±10% 机械摆动
- [ ] 利息、税项的处理已声明（计算 or 口径声明）
- [ ] 历史回测做了：用模型逻辑还原最近 1–2 个实际年度，分项偏差 <10%
- [ ] 基年/次年预测对照过一致预期和公司指引，偏离 >10% 写了分歧理由
- [ ] 配色规范：蓝=输入 / 黑=本表公式 / 绿=跨表引用
- [ ] recalc 零错误；data_only 读回关键输出与手算一致
- [ ] 币种、单位、财年口径已声明（财年非 12 月截止的专门标注）

## 阶段 3 估值与目标价

- [ ] 至少两种估值法（相对估值 + SOTP 或 DCF）
- [ ] 相对估值选了 5–10 家可比，给了分位统计（最大/75 分位/中位/25 分位/最小）
- [ ] 分部 SOTP（多业务公司）：每个分部估值方法和倍数有依据
- [ ] DCF（如做）：WACC 拆解、永续增长/退出倍数、敏感性表齐
- [ ] 目标价能从估值方法回溯出来，不是主观臆定
- [ ] 给了评级（首次）、上行空间、3–5 条催化剂
- [ ] 现价是结算日线收盘口径（不是盘中价），标了日期；市值 = 结算收盘 × 股本（股本口径绿鞋前后标清）
- [ ] 内部一致性校验过：`prev_close ×(1+涨幅)` = 结算 `close` 本身，没用涨幅反推 close
- [ ] 估值页并入阶段 2 的 Excel，没另起文件
- [ ] 估值用的每股数、利润数与阶段 2 模型一致

## 阶段 4 图表

- [ ] 40–70 张，覆盖阶段 1/2/3 数据 + 外部行情
- [ ] 必备图齐：股价走势对比、分部收入、盈利预测、SOTP/足球场、（若做 DCF）敏感性热力图
- [ ] 股价走势对比图双 Y 轴绝对价格、日频逐日真实收盘，未指数化/归一化、未插值
- [ ] 每张图配 `_data` 底稿，每个系列指得到来源；没有无源/示意/凑数系列
- [ ] 必备图缺数的已降级或省略并注明，没补造（少一张真图好过一张假图）
- [ ] 跑 `scripts/anti_fabrication_lint.py` 通过
- [ ] 直角坐标图无上/右边线、无网格线，X/Y 实线在
- [ ] 成对/可对照图按窄版（50% 栏宽）生成
- [ ] 中文不乱码（字体配置正确）
- [ ] 配色统一（一套主色）
- [ ] 每张 300 DPI，清晰不模糊
- [ ] 每张有标题（图表 N + 描述 + 单位）和来源脚注
- [ ] 连续编号无断号
- [ ] 打包成 zip + 图表清单

## 阶段 5 最终报告（DOCX）

- [ ] 25–45 页 / 1.5–3 万字 / 40–70 图全部嵌入正文
- [ ] 关键财务表 8–15 张
- [ ] 第 1 页齐：标题、代码、投资评级表（评级首次 / 当前价带日期 / 目标价 / 上行空间）、报告摘要（紧接评级表下方，开头带点题钩子）
- [ ] 第 2 页齐：盈利预测及估值表、基本状况、股价走势图（顺序对即可，自然流式分页）
- [ ] 没有独立"引言"段，正文从"一、公司概况"起
- [ ] 只有页码内容目录，没有单独的图表目录（list of figures）
- [ ] 正文是中式逻辑链结构，章名带判断
- [ ] 风险提示章节在（8–12 条）；保留评级和目标价
- [ ] 没有合规样板（评级体系图例、署名执业编号、免责声明）
- [ ] 每页基本都有图或表，版面 60–80% 有内容，无孤立标题、无单图占整页
- [ ] 图表标题在上、来源在下，连续编号
- [ ] 无占位句（"本章节将包括""此处插入图表""详见模型"）

## 阶段 5 · DOCX 格式契约（对照 `references/Word文档格式标准.md`）

- [ ] 字体：`w:rFonts` 的 eastAsia / ascii / hAnsi 三属性都是 Source Han Serif CN，仅常规+加粗
- [ ] 页面：A4 纵向；页边距 上1.8 / 右1.8 / 下1.6 / 左1.8 cm；页眉页脚距边界 0.8 cm
- [ ] 正文 10.5pt、行距 1.20 倍、段后 3pt、两端对齐；标题 H1/H2/H3 字号、颜色（#1F4E5F / #2F5965 / #333333）、段距合规
- [ ] 表格：真 Word 表格，每张单一 `tblW type="dxa"`（无 auto）；表头 #1F4E5F 底白字居中；首列左对齐、其余列居中；单元格边距 上下80/左右120 DXA
- [ ] 图片居中，宽度 ≤ 正文 85%；图注「图表 N：<主题>」9pt 加粗在图下；来源 7pt #666666 在图内、不在图注下方
- [ ] 成对图双列并列用无边框二列表格，左右各 50%、每格图居中
- [ ] 列表用真 numbering.xml（无假项目符号）；超链接在 .rels；正文无裸 URL
- [ ] 页脚是真 Word 页码域（非静态数字）
- [ ] 无禁止项：emoji、markdown 残留、纯英文名词堆砌标题、任何图表的网格线、直角坐标图的上/右边线
- [ ] **DOCX 渲染成 PDF/PNG 做视觉核验（LibreOffice 或 Word），强制——结构校验代替不了**

## 跨文件一致性（最容易出错，重点查）

- [ ] 营收数：研报正文 = Excel 模型
- [ ] 归母净利、EPS：研报 = 模型
- [ ] 各分部收入：研报 = 模型 = 图表
- [ ] 目标价、评级：封面 = 估值章 = 摘要
- [ ] 预测年份口径一致（哪年是 A、哪年是 E 全文统一）
- [ ] 口径一致性专项：同一指标全报告口径统一（招股书/年报、绿鞋前后、账面/经调整），关键数就近标了口径
- [ ] 高风险数（现价/市值/份额）验证换了异源，没用同源摘要或同盘中价 feed 互证
- [ ] 抽 10–15 个关键数，在 DOCX 和 Excel 之间核对一遍

## 写作（全文通用）

- [ ] 投资要点是"加粗判断 + 展开"
- [ ] 判断后跟数字和来源
- [ ] 无填充套话、无 -ing 伪分析尾巴、无机械三段式、无金句口号
- [ ] 句子节奏有长短变化
- [ ] 章节标题、小标题都带判断，非中性名词或翻译腔抽象词
- [ ] 无独立引言段；摘要开头有一句点题设问/钩子，正文直接从"一、公司概况"起
- [ ] 无"本文不讨论 / 不覆盖 X"式否定铺垫，直接正面写覆盖了什么（分析性否定不算）
- [ ] 无硬翻译 / 直译 calque（成本桥、顺风逆风、价值洼地、敞口、硬约束），行话（护城河、对标）保留
- [ ] 无装饰性比喻（删掉信息不减的，船票、埋下隐忧、压舱石）
- [ ] 无情绪化 / 暴力词（暴雷、杀估值、吃掉、蛋糕在缩），换成了字面说法
- [ ] 对冲词没堆砌（可能、或许、某种程度上一句留一个），数据的"约、近"保留
- [ ] 同一个东西从头到尾用同一个词，没换着叫
- [ ] 并列枚举用 bullet / 分段（每项加粗领起），连贯论证用散文，没塞成流水账也没拆碎
- [ ] 判断 / 风险都追了一层归因（机制 + 落点），不停在现象
- [ ] 总检查：把形容词、比喻、对仗拿掉后还剩具体信息，不是空话；逐句拿不准就查 `去AI味案例库.md`
