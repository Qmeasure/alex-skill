# 报告模板与版式

这份规定最终 DOCX 的版式：封面、盈利预测及估值表、图表编号、字体配色、章节骨架。按中文券商深度报告的惯例处理。组装报告（阶段 5）前先读这份。

## 封面

封面是读者最先看到、信息密度最高的部分。按两页组织：第 1 页是结论与摘要，第 2 页是数据与图。**分页按内容自然流式，不插硬分页符——"第 1 页 / 第 2 页"是版面意图而非强制断页；摘要过长时压缩摘要，别为凑分页打乱顺序。**

### 第 1 页（从上到下）

1. **标题区**。主标题是一句带判断的话（"守正创新，基业长青""国产 AI 芯片领军者，云边端共铸核心壁垒"），副标题是"[公司]首次覆盖"或"[公司]深度研究报告"。主标题大字加粗。
2. **股票标识**。公司全称 + 代码（腾讯控股 00700.HK / 寒武纪 688256），加行业归类（社交Ⅱ / 集成电路）。
3. **投资评级表**。一个框里给四项：**评级（首次）**、**当前价**（标日期，如"股价为 2025 年 12 月 11 日收盘价"）、**目标价**、**上行空间**（=（目标价 − 当前价）/ 当前价）。评级用所在市场惯用的档（A 股：买入/增持/中性/减持/卖出；港股美股可用 Buy/Hold/Sell 或对应中文）。
4. **报告摘要**。紧接评级表下方。**不单独写引言**——开头用一句点题设问或钩子领起（这家公司最值得研究的那个问题，原引言那句设问移到这里），随后按"一 / 二 / 三 + 投资建议 + 风险提示"分层写。这是全文最浓缩的版本，每层一句加粗判断领起。结构对应正文逻辑链：概况 / 空间 / 驱动，最后给投资建议（盈利预测数 + 评级）和风险提示。报告没有独立引言段，摘要之后正文直接从"一、公司概况"起。

### 第 2 页（从上到下）

5. **公司盈利预测及估值表**。核心表，横轴是年份（含 A/E 标识），纵轴是关键指标：

   | 指标 | 2023A | 2024A | 2025E | 2026E | 2027E |
   |------|-------|-------|-------|-------|-------|
   | 营业收入（亿元） | | | | | |
   | 增长率 yoy% | | | | | |
   | 归母净利润（亿元） | | | | | |
   | 增长率 yoy% | | | | | |
   | 每股收益（元） | | | | | |
   | PE | | | | | |
   | PB | | | | | |

   表下标注口径，如"股价截止 [日期] 收盘价，每股指标按最新股本摊薄，EPS、PE 采用经调整利润计算"。

6. **基本状况**。总股本、流通股本、市价、总市值、流通市值。
7. **股价走势对比图**。个股 vs 指数（沪深300 / 恒生指数 / 标普500），近 12 个月。**双 Y 轴绝对价格、日频逐日真实收盘**（不插值），细则见 `references/阶段4-图表生成.md`。这是阶段 4 的必备图之一。

封面不放分析师署名、执业编号、评级体系图例、免责声明这些合规样板（本 Skill 省略）。但评级、目标价、风险提示要保留，它们是分析结论。

## 图表编号与来源脚注

中文研报把图和表统一编号，叫"图表 N"，全文连续不断号（图表 1、图表 2、图表 3……不分图和表两套号）。深度报告通常 40–70 个。

每个图表的规范：

- **标题在上**：「图表 N：[描述性标题]」，比如「图表 11：腾讯分业务营业收入（亿元）」。标题带单位。
- **来源在下**：小字，「来源：公司公告、WIND、XX证券研究所」。
- 只保留页码内容目录，不另做图表目录（list of figures）。图表全文连续编号，但不在正文开头列一份图表清单。

## 正文章节骨架

正文按中式投资逻辑链组织，不按固定清单。骨架是这套，具体章名按公司和行业起得有判断、有画面（见 `研报写作规范.md`）：

```
一、公司概况：[一句判断式标题]
  1.1 历史复盘——沿革、关键节点、股价复盘
  1.2 业务布局——分部拆分、变现模式、股权与治理、管理层
  1.3 财务分析——历史收入/利润/现金流/回报的趋势

二、行业空间：[一句判断式标题]
  2.x 市场规模、增长驱动、政策、产业链位置
  （可按业务分部分别讲空间，如 2C / 2B）

三、核心驱动 / 竞争壁垒：[一句判断式标题]
  3.x 支撑投资逻辑的关键变量——技术壁垒、生态、新业务、AI 等
  这是全文最重的一章，回答"凭什么是它、凭什么是现在"

四、盈利预测与估值
  4.1 盈利预测——分部收入预测 + 利润预测 + 关键假设
  4.2 估值分析——相对估值 + SOTP + DCF 辅证 + 目标价 + 评级 + 足球场图
  4.3 风险提示——8–12 条，分类列

附：盈利预测三表摘要（可选）
```

章节顺序可随公司调整（比如华创-寒武纪把"竞争壁垒"放在行业之后、盈利预测之前），但逻辑链的走向不变：先认识公司，再看行业给了多大空间，再论证它凭什么吃下这个空间，最后落到能赚多少、值多少、风险在哪。每一章都在推进投资论点，不是各自独立的资料堆砌。

骨架里 1.1 / 4.1 这种子项名只是结构占位，**落到报告里的小标题要带判断，不要停在中性名词**（详见 `研报写作规范.md`）。比如"投入端：capex 总量与节奏"这种中性标题，改成"capex 高度集中于四家超大厂，门槛把中小玩家挡在外面"这类带观点的。大标题小标题一视同仁。

## 字体、配色、版式

字体、字号、行距、页边距、标题颜色、表格、图注这些正文排版数值，统一以 `references/Word文档格式标准.md` 为准（思源宋体 Source Han Serif CN、正文 10.5pt、A4、上1.8/右1.8/下1.6/左1.8 cm 等），这里不另列一套，避免冲突。本文件只管研报特有的部分：

- **图表配色**：可贴券商风格（中泰红系、华创蓝系），全报告一套主色。这只作用于阶段 4 生成的图表内容，与格式标准里的标题文字颜色是两回事。图表配色见 `references/阶段4-图表生成.md`。
- **版面密度**：正文 60–80% 版面有内容（图文表混排，不要大片留白），尽量每页都有图或表。避免"一页就一张图"或"标题下面没正文"。图配文、表配解读。
- **双图并列**：成对/可对照的图（分部增速对比、可比公司倍数、三情景、毛利率 vs 净利率等）鼓励一行两图各占 50% 居中并列，提升信息密度；不要永远一行一图。单图占整页仍禁止。实现细节（窄版生成、无边框二列表格）见 `references/阶段4-图表生成.md` 与 `references/Word文档格式标准.md`。

## 体量门槛

- 25–45 页
- 1.5–3 万字中文正文
- 40–70 个图表全部嵌入（不是附在文件夹里）
- 关键财务表 8–15 张

达不到这些，说明内容没有充分展开，补充完善后再交付。
